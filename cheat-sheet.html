Intro to Git Commands

Git is a version control software system designed for facilitating GitHub activities. Version control is important for tracking changes and preventing loss of data. This cheat sheet provides common git commands for reference.

Making Changes

These commands are used to review edits and changes to be made to the repository.

git status

Shows files to be commited.

git add

Stages the changes to be committed.

git commit -m [description]

Records changes permanently to version history.

git push

Uploads all local branch commits to GitHub.

git diff

Shows changes not currently staged to be committed.

git reset [file]

Unstages a file from being committed, but preserves its content.

Group Changes

These commands are for accessing and changing branches.

git branch

Lists all branches in the current repository.

git checkout [branch name]

Switches to specified branch and updates working directory.

git merge [branch]

Combines specified branch's history into current branch.

git branch -d [branch name]

Deletes specified branch.

git branch [branch name]

Creates a new branch.

Create Repositories

These commands are for creating and changing repositories.

git init [project name]

Creates new local repository with specified name.

git clone [URL]

Downloads a project and its entire history from the specified URL.

Refactored Filenames

These commands are for changing and removing files.

git rm [file]

Deletes file from working directory and stages file for deletion.

git rm --cached [file]

Removes file from version control but keeps file locally.

git mv [file original] [file renamed]

Changes file name and prepares for commit.

Review History

These commands are for inspecting the evolution of project files.

git log

List history for current branch.

git log --follow [file]

Lists version history for specified file including renames.

git diff [first branch]...[second branch]

Shows content differences between two branches.

git show [commit]

Shows meta deta and content changes of a specific commit.

Redo Commits

These commands are for erasing mistakes and changing history.

git reset [commit]

Undo all commits after specified commit.

git reset --hard [commit]

Discards all history and reverts back to specified commit.
