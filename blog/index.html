What is the Enumerable Module?

The Enumerable Module, is a module that you can mix in. OK, let’s stop right there. Have I already lost you? Let’s trying breaking it down into smaller components.

First of all, what is a module? A module is a toolbox that contains set methods and constants. There are so many Ruby tools in the toolbox that you will want to use, but if Ruby loaded them all every single time it ran, then it would become cluttered. Besides, even though you will want to use each tool, you will be using them at different times. Therefore, we keep them in modules and only pull out the module toolboxes when we need to use something stored inside of it!

That makes sense. So what is a mixin? That is when you mix together the behaviors of a module and a class. When a module is used to mix additional behavior and information into a class, it’s called a mixin. Coming back full circle to the Enumerable Module, can you guess what it is now? It’s a bunch of methods that work with collections. It’s essentially just a giant toolbox that we can take tools out of and use at our leisure!

The Enumerable Module empowers mostly the Array and Hash classes. You can read more about those here.


Stop Writing “For” Loops

A class that uses or “mixes in” the Enumerable Module must define the #each method. The #each method looks like this:


[1, 2, 3, 4, 5].each { |n| puts n }


This method is surprisingly straightforward once the syntax is explained. For each number in the Array, I named the number “n” (But you can name it whatever you would like… “number” or “days_without_you” or anything!) and then I put “n”. This will simply list out each number:


1 2 3 4 5


You can manipulate “n” in a bunch of ways, depending on what you are trying to do. With each element in the array, it will execute the given block. The Enumerable Module is a toolbox of methods building upon this method. Just to name a few: all?, any?, collect, detect, each_cons, each_slice, each_with_index, enum_cons, enum_slice, enum_with_index, find, find_all, grep, include?, inject, map, max, member?, min, partition, reject, select, sort, sort_by, to_a, to_set, zip. Phew!


#map

Let’s start with an easy method, #map. This method is extremely similar to the #each method, but with one big difference. #each simply iterates over the given enumerable, running the block for each value. It dismisses the return value of the block after the method is complete. For example:


[4, 5, 6].each { |x| x + 1 }
# => [4, 5, 6]


On the other hand, #map returns a new object with the changes that you make inside of the block.


[1, 2, 3].map { |i| i + 1 }
# => [2, 3, 4]


Therefore it “maps” each element to a new one. It is also worth noting that #map doesn’t set an values in the original collection, but only in the returned copy of it. Also, #map and #collect are exactly the same! I have seen #map used more in practice, so I would recommend sticking with that one, but it doesn’t really matter too much.


#group_by

Can you guess what this method does just by seeing the name? That’s the entire point of Ruby – to be a programmer-friendly language! The fact that it is so easily readable is what makes it great. Let’s check out an example of #group_by:


(1..10).group_by { |i|i % 3 } # => {1=>[1, 4, 7, 10], 2=>[2, 5, 8], 0=>[3, 6, 9]}


As you can see, this method grouped each of our numbers according to their divisibility by 3. It groups the collection by the result of the block, and returns a hash where the keys are the evaluated result from the block, and the values are arrays of elements that correspond to the key.


#cycle

The last method in the Enumerable Module that we will look at today is #cycle. This method calls the block for each element repeatedly for “n” times, or forever is no time was given. If a negative integer is given or the collection is empty, then it does nothing.


array = ["a", "b", "c"] array.cycle { |x| puts x } # a, b, c, a, b, c,.. forever. array.cycle(2) { |x| puts x } # a, b, c, a, b, c.


And that’s it! If you have some extra time to spare, go ahead and check out the Ruby Docs on the Enumerable Module and try out some more methods!
