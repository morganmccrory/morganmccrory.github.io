Looping in Ruby vs. Looping in JavaScript

Looping in JavaScript is extremely similar to looping in Ruby. The only thing that changes is a little bit of syntax. Let's take a look at some specific loops in both languages.

The 'For' Loop

The 'for' loop is one of the most common loops in both languages. It is most useful for when you know how many times you'll be looping. Instead of writing "puts 1, puts 2, puts 3, puts 4, puts 5", we can write a loop that does this for us.



Here is a 'for' loop in Ruby:

for num in 1..5
  puts num
end

# returns
1
2
3
4
5

In this example, each number 1 through 5 was printed on a new line to the console. This code is telling Ruby that for the variable "num" in the range of 1 to 5, do what is next. In this case, "puts num" was what was next. One of the biggest tings to note is that we used an inclusive range in this example with the use of the two dots in 1..5. If we had used an exclusive range instead, written as 1...5, then it would have only "puts"ed out numbers 1 through 4.



Let's look at the same 'for' loop in JavaScript:

for(var counter = 0; counter < 6; counter++) {
  console.log(counter);
}

This does the exact same thing in JavaScript that it did in Ruby - it puts a number 1 through 5 (inclusive) on each line in the console. However, there is some new syntax here, so let's take a look:



var counter = 1;


This declares a variable named "counter" with a value of 1. When the 'for' loop is executed, the code in the code block (the part between { and }) - it does so by starting off where counter = 1.



counter < 6;



This tells the loop to continue running while counter < 6. So when counter = 2 or counter = 3, the loop will continue to run. However, once the counter is no longer less than 6, it will stop running.



counter++



This is simply a shorthand way of saying "counter = counter + 1". Therefore, each time the loop runs through the code block, the value of counter will increase by one. This is what makes it possible for our loop to end so we don't end up creating an infinite loop.



'While' You're Here...

Let's now take a look at 'while' loops in both Ruby and JavaScript. 'While' loops are useful when you want to repeat a certain block of code while a certain condition is true, but you aren't sure how many times it will be repeated. One example is when you are prompting a user for a zip code, which is 5 integers, and they give you a 6 digit number. You may have to re-ask them several times before you get the input that you are looking for. To do this, we use a 'while' loop. It checks to see if a certain condition is true, and keeps running as long as it is true.



Here is a Ruby 'while' loop:



counter = 1
while counter < 6
  puts counter
  counter += 1
end

This code should be fairly self-explanatory. We define a variable named "counter" and set it equal to 0. While the counter is less than 6, put the value of the counter. Also, each time the loop runs, add 1 to counter and set that as the new value of counter.



Now let's look at that same code in JavaScript:



counter = 0;

while(counter < 6) {
  console.log(counter);
  counter++;
}

As you can see, the 'while' loop still evaluates a condition and continues running as long as the condition is true. The code above demonstrates the same example that we saw with Ruby, but you can also evaluate a condition without using numbers:



var condition = true;

while(condition) {
  console.log("The condition is true!");
  condition = false;
}

Notice the "while(condition)" syntax in the above example. This is a shorthand way of saying "while condition === true". It is just shorter and easier to leave out the "=== true" portion and leave it as implied.



Beware of infinite loops! Notice that inside of the loop, we change the condition to equal false. This is very important, because otherwise we would be creating a loop that would never end. This will cause your console to crash and is a mess all around. Make sure that you never create an infinite loop!
