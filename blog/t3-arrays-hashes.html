Ruby: Introduction to Arrays and Hashes

You can use an array to store a list inside a single variable. The list can be of anything – numbers, strings, symbols, etc. They don’t have to be in any order, but each element in the array is assigned a number. This is called the index. What is interesting is that instead of beginning with "1", the first object in an array is actually "0". From there, the index goes in ascending order to 1, then 2, then 3, etc. This can take some getting used to! We can access each element of the array through the index number:

array = [3, 8, 19, 42, 7, 35] array[3] # returns a value of 42


The example above shows how the index works for our array. The index numbers for each of the elements are: 3 has an index of 0, 8 has an index of 1, 19 has an index of 2, etc. This is called access by index. We can access the “n”th element of an array by putting the index number in square brackets (as seen above). What if we switched the order around?

array = [19, 7, 42, 35, 3, 8] array[3] # returns a value of 35


While order does not really matter in an array, it does matter when assigning index numbers to each element. Because the order of the array changed, the value of index 3 changed. Now, what if we call an index that doesn’t exist?

array = [19, 7, 42, 35, 3, 8] array[7] # returns nil


Nothing gets returned because index 7 does not exist! As I mentioned earlier, arrays don’t have to be just numbers. They can be strings, boolean values, or (almost) anything!

array_of_strings = [“Ruby”, ”is”, “my”, “favorite”, “language!”]


We can also create a multidimensional array. This is simply an array of arrays. It has this name because when you add arrays as elements to another (main) array, it expands the shape of the main array to look more like a box. Yes, programmers like to keep naming conventions fairly simply!

Introduction to Hashes

OK, so we just talked about arrays and the index. But what if we want to create our own index and have it be something other than numbers? An array has the potential to be very long, and it would be very difficult to remember the element that is at index 227. Ruby has a solution! That is where hashes come in.

Very similar to JavaScript objects or Python dictionaries, a hash is a collection of key-value pairs. A hash is written like this:

hash = { “dog” => “Old Yeller”, “cat” => “Garfield”, “owl” => “Hedwig” }


In this example, the animal types (dog, cat, owl) are the keys and the names (Old Yeller, Garfield, Hedwig) are the values, respectively. Creating a hash this way, by giving it a name and setting it equal to key => value pairs, is called literal notation. But you can also create an (empty) hash by entering: hash_name = Hash.new

Much like with arrays, we can call on a value of a hash by typing this:

hash[“dog”] # returns “Old Yeller”


If we used literal notation to create the hash, then we can add to it directly. However, if we used Hash.new to create the hash, then we can add to it like this:

nfl = Hash.new nfl[“Bears”] = “Chicago”


And voila! We have now added the “Bears” key and “Chicago” value to the nfl hash.

It’s A Symbol

But the fun doesn’t stop here! I want to take a moment to address the slightly more advance topic of symbols. A symbol is another type of Ruby object. They are not strings! There can only be one copy of a symbol at any given time and symbols cannot be changed. The syntax looks like this:

:symbol


So what can we use symbols for? Incidentally, they make great keys for those hashes that we just learned about! One of the biggest benefits is that hash lookup is faster with symbol keys than with string keys. As of Ruby 1.9, the syntax for symbols changed, as did the method for using them within hashes. The most current version if using symbols as keys, would be like this:

my_hash = { one: 1, two: 2, three: 3, four: 4 }

The colon now goes at the end of the symbol, and the hash rocket (=>) is no longer needed! It looks much cleaner this way, don’t you think? I hope this was a helpful guide about arrays, hashes, and symbols!
