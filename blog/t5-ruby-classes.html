Getting an “A” in Ruby Classes

I was talking with some friends about Ruby last night, and I heard the perfect analogy to describe Ruby classes. Let’s pretend that you want to create some widgets. They will all have the same shape and basic characteristics. It would be so cool if you could create a machine that can “stamp” each widget out – in fact, that would probably save you a lot of work. That is exactly what a class does. It allows you to create your own specific “stamp”, which can then create different objects as if on an assembly line. Each new object, by default, has the attributes and methods that you specified built into them.



Almost everything in Ruby is an object. We’ve been using them all along! That’s why Ruby is an object-oriented programming language, because the things that we are manipulating with our methods are called objects!



To illustrate, let’s build our own class right now. First, we must give the class a name. Proper formatting for class names start with a capital letter and use CamelCase as opposed to_underscores.

class Dog
end


So far, our class does absolutely nothing. That’s right – nada. Since we want our class to actually be functional, let’s add the initialize method. This method is required for every class that you make. It is what Ruby uses to “start up” each object that the class creates.

class Dog
    def initialize
    end
end


Now that we’ve defined our initialize method, we need to assign instance variables by stating them inside this method. An instance variable is a variable that is bound to an instance of a class. We signify that a variable is an instance variable by using the @ symbol before the variable name. What instance variables should our Dog class take?

class Dog
    def initialize(breed)
    @breed = breed
    end
end


Now, we are ready to create an object! Let’s create our first instance of the Dog class.

class Dog
    def initialize(breed)
    @breed = breed
    end
end

snoopy = Dog.new(“beagle”)


This looks great, but our class doesn’t really do anything. It lets us create a new object and stores what breed it is, but maybe we can make it a little more interesting. If we add another method to our class, the objects we create will be able to use that method.

class Dog
    def initialize(breed)
    @breed = breed
    end

    def bark
        puts “Ruff! Ruff!”
    end
end

snoopy = Dog.new(“beagle”)
snoopy.bark

# returns “Ruff! Ruff!”


Now every object in the Dog class that we make can call the bark method. We can put as many methods into the class as we would like – but be careful to not overcrowd it. You should only put in methods that all (or most) of the objects that you create will use.
